# 决策池（中文版对照）

这是一个持续维护的交易决策记录，用来复盘每一次合约判断是否有效。它不是为了证明过去判断正确，而是为了提高下一次判断胜率。

## 核心原则：决策池也不能无限膨胀

决策池如果每次都把完整历史塞进上下文，会造成两个问题：一是信息噪音太多，二是容易被过去判断锚定。因此维护规则如下：

- **活跃仓位放最上面**：只保留当前仓位、最近一次主判断、目标、止损/失效、下一次检查时间。
- **最近 5-10 条关键决策保留在主文件**：方便看连续推理链条。更早的决策应进入月度归档，只在复盘时读取。
- **复盘结果比原始聊天更重要**：每次决策后要记录 24h、72h、7d 结果，最大有利波动、最大不利波动、是否触发目标/止损，错误属于宏观、数据、品种选择、时机、杠杆路径还是事件理解。
- **不能为了连续性维护旧观点**：每次新判断都要从实时价格、宏观、衍生品、情绪、事件重新计算。历史判断只是“上一次模型状态”，不是新结论的证据。
- **合约语境下使用明确动作**：用户主要做合约，所以记录动作时优先写 hold / close / switch / flip long / flip short / no trade，而不是模糊的“减仓一点”。

## 合约决策动作定义

| 动作 | 含义 | 适用场景 |
|---|---|---|
| hold long | 继续持有多单 | 趋势和宏观仍支持，未触发失效位 |
| close long | 平掉多单 | 上涨胜率下降，或者事件风险不值得继续承受 |
| flip short | 平多并反手做空 | 价格结构破位，同时宏观/衍生品确认空头更占优 |
| switch product | 换品种 | 当前方向可做，但持有标的相对强弱差，例如 ETH/SOL 弱于 BTC |
| no trade | 不开仓 | 事件临近、上下插针概率大，胜率不足以覆盖合约风险 |
| hold short | 继续持有空单 | 下跌结构未修复，反弹无法站回关键位 |
| close short | 平掉空单 | 空头拥挤、价格收回关键位、宏观转向风险偏好修复 |

## 决策时必须记录的因子

- 当前价格、时间和来源。
- 持仓方向、入场价、杠杆、强平价，如果用户提供。
- BTC 方向结构：BTC 是加密市场主锚，山寨币不能脱离 BTC 判断。
- 相对强弱：目标币相对 BTC、ETH、SOL 是否强。
- 资金费率：多头或空头是否拥挤。
- 未平仓合约 OI：价格上涨/下跌时 OI 如何变化，判断是主动增仓还是被动爆仓。
- 爆仓热力图 / 清算集中区：哪里容易被插针。
- 标记价和指数价：合约爆仓和资金费率通常不是只看最新成交价。
- 订单簿深度：薄流动性下更容易被拉爆或砸穿。
- VIX、美元指数 DXY、美债收益率、实际利率、油价。
- ETF 净流、稳定币供应、链上交易所流入流出。
- 未来 24-72 小时事件：FOMC、CPI、PPI、PCE、非农、期权交割、重大讲话、地缘突发。

## 用户偏好规则

- 用户需要一个综合评估后胜率最高的主操作，不要宽泛中立。
- 用户做合约，所以“减仓”通常不如“继续持有 / 全平 / 换标的 / 反手”清晰。
- 概率可以是主观概率，但必须绑定到目标、失效位和事件节点。
- 不要把宽区间当成结论。区间可以作为止盈区或止损区，但主判断必须明确。
- 每次都要客观重估，不能为了维护过去观点而忽略新信息。
- 默认关注 BTC/ETH/SOL，除非用户明确要求分析其他币种或特殊产品。
